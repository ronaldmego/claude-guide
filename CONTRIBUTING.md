# Contribuir

Kit personal y mínimo en español para usar **Claude Code** con repositorios,
código o datos.

## Formas de contribuir

- Mejorar las plantillas (`CLAUDE.md`, `settings.json` o la skill de ejemplo).
- Corregir errores técnicos, enlaces rotos o explicaciones ambiguas.
- Simplificar el quick start sin convertir el repo en una enciclopedia.

## Reglas

- **Todo en español** (es el nicho); deja en inglés solo los términos técnicos naturales
  (track, cowork, skills, hooks, etc.).
- Prefiere fuentes **oficiales / first-party** y enlaza sólo las imprescindibles.
- **Genérico y reutilizable** — sin secretos, sin datos personales, sin contenido de un
  empleador. Las plantillas van sanitizadas (placeholders, nunca valores reales).
- **Menos es más** — una contribución debe reducir ambigüedad o riesgo; no agregar
  catálogos, listas de herramientas o documentación preventiva sin un caso real.

## Cómo

1. Abre un issue describiendo el cambio.
2. Haz fork, crea una rama, aplica el cambio.
3. Abre un PR referenciando el issue. Un PR = una cosa.
